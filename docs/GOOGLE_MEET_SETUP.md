# Google Meet link (without sending guests a Google Calendar invite)

## How it works in this project

1. After a customer pays (or completes a complimentary booking), your server can call `createGoogleMeetEventForBooking` in `src/lib/googleCalendar.ts`.
2. That function uses the **Google Calendar API** to insert an event on **your** calendar with `conferenceData` so Google returns a **Meet URL** (`hangoutLink` / video `entryPoint`).
3. The event is created with **`sendUpdates: 'none'`**, so Google does **not** email the customer a calendar invitation. You control what you send (e.g. only the Meet link in your own confirmation email).
4. The calendar event is mainly the supported way to **obtain a stable Meet link** programmatically. There is no separate “Meet-only” REST API that replaces this for arbitrary accounts without Calendar access.

### Virtual Session and other paid bookings

- **Virtual Session** (1:1) and **Group cohort** (scheduled group) use the same pipeline: after **Stripe** reports successful payment, the **Stripe webhook** calls `fulfillBookingMeetAndEmails`, which creates the Meet link (if Google Calendar API + OAuth are configured) and sends the **confirmation email** with the join link.
- **Complimentary** nonprofit requests use the same Meet creation path after the booking is confirmed server-side.
- If Meet creation fails, emails still send but may say the link is pending — check server logs and `docs/GOOGLE_MEET_SETUP.md` (OAuth, Calendar API, `GOOGLE_REFRESH_TOKEN`).

## Microsoft Teams

Teams meeting URLs are **not** generated by this codebase. Options:

- Paste a Teams **join link** manually into the customer’s confirmation email, or  
- Add a Teams link in the booking **notes** field and include it in your email template.

## Step-by-step: OAuth (recommended for `primary` calendar)

1. In [Google Cloud Console](https://console.cloud.google.com/), create a project (or pick one).
2. Enable **Google Calendar API** for that project.
3. Configure **OAuth consent screen** (External or Internal as appropriate).
4. Create **OAuth client ID** → type **Web application**.  
   - Add authorized redirect URI: your app’s callback, e.g. `https://yourdomain.com/api/google/oauth/callback` (and `http://localhost:3000/...` for local dev if you use it).
5. Install deps are already in the repo; set environment variables (see `.env.example` or your deployment):

   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - Run your OAuth “connect Google” flow once to obtain a **`GOOGLE_REFRESH_TOKEN`** and store it securely.

6. Set `GOOGLE_CALENDAR_ID` if not using `primary` (default is the authenticated user’s primary calendar).

7. Redeploy / restart `npm run dev` and complete a test booking with calendar integration enabled in your webhook flow.

## Alternative: service account

1. Create a **service account** in the same Google Cloud project, enable Calendar API.
2. Create a JSON key; set `GOOGLE_CLIENT_EMAIL` and `GOOGLE_PRIVATE_KEY` (with `\n` escaped newlines in env).
3. **Share** the target Google Calendar with the service account email (e.g. “Make changes to events”) — service accounts do not own a consumer Gmail calendar without sharing.

## Env summary

| Variable | Purpose |
|----------|---------|
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` / `GOOGLE_REFRESH_TOKEN` | OAuth user calendar |
| `GOOGLE_CLIENT_EMAIL` / `GOOGLE_PRIVATE_KEY` | Service account JWT |
| `GOOGLE_CALENDAR_ID` | Calendar ID (default `primary`) |

## Troubleshooting

- **No Meet link**: ensure `conferenceDataVersion: 1` and Workspace / project allows Meet creation; check API error logs.
- **Wrong time**: preferred times are interpreted in the booking timezone (e.g. `America/Chicago`).
